# 🎲 骰子酒馆 · KCD2 Farkle

> *"掷出你的命运，或在豪赌中一无所有。"*  
> ——某位在库特纳霍拉输光裤子的旅人

《天国拯救2》（Kingdom Come: Deliverance 2）酒馆骰子小游戏 **Farkle** 的线上双人复刻版。  
无需账号、无需服务器，基于 P2P WebRTC 技术，打开浏览器即可与朋友一较高下。

---

## ✨ 特性一览

- 🌐 **零服务器联机** — 借助 [Trystero](https://github.com/dmotz/trystero) + WebRTC + Nostr 去中心化信令，无任何后端依赖
- 🔐 **公平掷骰** — Commit-Reveal 密码学承诺协议（SHA-256），杜绝作弊
- 🎲 **35 种骰子** — 忠实复刻原游戏全部骰子，各有奇特权重
- 🃏 **双重选骰模式** — 自由模式 & 轮抓草稿模式，斗智斗勇
- ✂️ **猜拳决先手** — 石头剪刀布决定行动顺序（同样使用 Commit-Reveal 保证公平）
- 💬 **酒馆旁白系统** — 20+ 位 NPC 实时起哄，沉浸式中世纪氛围
- 🎉 **三档粒子庆祝** — 高分、顺子、满盘各有专属烟花特效
- 😈 **恶魔之首（百搭）** — 特殊骰子掷出百搭面，贪心算法自动求最优得分

## 🕹️ 如何联机对战

> 已部署的 GitHub Pages 地址：**`https://<你的用户名>.github.io/KCD2-Farkle/`**

### 方式一：分享房间链接（最方便）

1. **房主**打开上方网址，点击「**创建房间**」
2. 系统自动生成 **6 位房间码**，点击「复制链接」将完整链接发给对方  
   *(链接形如 `https://xxx.github.io/KCD2-Farkle/?room=AB3C7F`)*
3. **客人**直接点开链接，输入昵称后自动进入房间
4. 双方连接成功后，房主配置规则，点击「**开始游戏**」

### 方式二：手动输入房间码

1. **房主**打开网址 → 「创建房间」→ 记下显示的 6 位大写房间码
2. **客人**打开同一网址 → 「加入房间」→ 输入昵称 + 房间码 → 「加入」
3. 等待房主点击「开始游戏」

> **注意**：联机基于浏览器 WebRTC，两端均需能正常访问互联网。  
> 国内环境若遇连接失败，可尝试挂代理后重试。

## 🎮 游戏规则

### 目标

率先将总分累积到**目标分数**（默认 4000 分，可在开局前调整为 500–10000）的玩家获胜。

### 一回合的流程

```
掷骰 → 选择得分骰子（至少选1枚）→ 再次掷骰 / 入账
```

| 操作 | 说明 |
|------|------|
| **掷骰** | 掷出所有未锁定的骰子 |
| **选择骰子** | 点击得分骰子将其锁定，选中的骰子**必须全部参与得分** |
| **继续掷骰** | 锁定部分骰子后，对剩余骰子再次下注 |
| **入账（Bank）** | 主动收手，将本轮积分加入总分，交给对手行动 |
| **爆点（Bust）** | 掷出的骰子中**无任何得分**，本轮积分清零，强制换手 |
| **满盘（Hot Dice）** | 6 枚骰子全部被锁定，重新掷全部 6 枚，积分继续累积！ |

### 得分表

| 组合 | 分值 |
|------|------|
| 单个 **1** | 100 分 |
| 单个 **5** | 50 分 |
| 三个相同（非1）| 面值 × 100（如三个4 = 400分）|
| 三个 **1** | 1000 分 |
| 四连 | 三连基础 × 2 |
| 五连 | 三连基础 × 4 |
| 六连 | 三连基础 × 8 |
| 小顺子（1-2-3-4-5）| 500 分 |
| 中顺子（2-3-4-5-6）| 750 分 |
| 大顺子（1-2-3-4-5-6）| 1500 分 |

> 😈 **恶魔之首**：魔鬼骰子掷出面值 1 时，变为**百搭**（face = 0），自动以最有利的面值参与计分，但不计入顺子判断。

## 🛠️ 技术栈

| 层级 | 技术 |
|------|------|
| UI 框架 | [Svelte 5](https://svelte.dev)（Runes 语法：`$state` / `$derived` / `$effect`）|
| 语言 | TypeScript（严格模式）|
| 构建工具 | Vite 8 |
| P2P 网络 | [Trystero](https://github.com/dmotz/trystero) + WebRTC + Nostr 信令 |
| 密码学 | 浏览器原生 WebCrypto API（SHA-256 Commit-Reveal）|
| 随机数 | Knuth MMIX LCG（BigInt 实现，确保双端一致）|
| 状态管理 | Svelte writable / derived stores |
| 动画 | 纯 CSS（火炬闪烁、粒子下落、气泡滑入）|
| 骰子渲染 | SVG 点阵 |
| 测试 | Vitest（计分逻辑单元测试）|
| 部署 | GitHub Pages（纯静态，零服务端）|

## 📦 部署到 GitHub Pages

### 自动部署（推荐）

1. Fork 本仓库到你的 GitHub 账号
2. 进入仓库 **Settings → Pages**
3. Source 选择 **GitHub Actions**
4. 在仓库根目录创建 `.github/workflows/deploy.yml`，内容如下：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm install
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist
      - id: deployment
        uses: actions/deploy-pages@v4
```

5. 推送代码后，GitHub Actions 自动构建并发布
6. 访问 `https://<你的用户名>.github.io/KCD2-Farkle/` 即可游玩

> **vite.config.ts 已预配置**：生产构建时自动将 `base` 设为 `/KCD2-Farkle/`，无需手动修改。
